## The Problem

Every agent session starts from zero.

Your agent re-discovers why that architectural decision was made. It re-learns which approaches already failed. It re-figures out which constraints are non-negotiable. Another engineer's agent did the exact same thing last week.

Individual agent memory is solved. What happens when agents need to *agree on what's true across a team* — that's still broken. That's what Engram fixes: a shared, persistent knowledge base that survives across sessions, syncs across teams, and catches it when two agents develop contradictory beliefs.

We're open source, MCP-compatible, and install in a single command. Apache 2.0.

## What We're Building Together

Engram is early-stage and moving fast. The codebase is small by design — lean, focused, and built around one hard problem. Right now the core is working. What's next:

- **Better conflict detection** — semantic similarity models that catch contradictions before they compound
- **Richer lineage** — full fact provenance, diffable memory history, rollback
- **Performance under real team load** — smarter indexing, smarter caching, smarter queries
- **Ecosystem integrations** — more IDEs, more agent frameworks, smoother install flows

Every one of these is a real technical problem. None of them is busy work.
